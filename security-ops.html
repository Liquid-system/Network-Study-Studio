Security & Operations / Hardening & Management

作ったネットワークを、守って運用する。

実務のネットワークエンジニアは、構築した後のほうが長い時間を過ごします。この単元では、機器に入る経路を守る設定(パスワード・SSH・ポートセキュリティ)と、障害対応に欠かせない記録の仕組み(Syslog・NTP)を学びます。

Device Hardening

管理アクセスを守る: パスワードとSSH

Telnet は通信が平文で流れるため、経路上で盗聴されるとパスワードがそのまま見えてしまいます。実務ではリモート管理を SSH(暗号化)に限定するのが基本です。

パスワードの基本

R1(config)# enable secret class123 R1(config)# service password-encryption R1(config)# banner motd #Authorized access only#
  • enable secret は特権モードのパスワードをハッシュ化して保存(enable password より安全)
  • service password-encryption は設定ファイル内の平文パスワードを暗号化表示にする
  • banner motd はログイン時の警告文。不正アクセスへの法的な牽制になります

SSHの有効化(順番が大事)

R1(config)# hostname R1 R1(config)# ip domain-name study.local R1(config)# crypto key generate rsa The name for the keys will be: R1.study.local How many bits in the modulus [512]: 1024 R1(config)# username admin secret cisco123 R1(config)# line vty 0 4 R1(config-line)# login local R1(config-line)# transport input ssh

RSA 鍵は「ホスト名 + ドメイン名」から作られるため、hostname と ip domain-name を先に設定しないと鍵を生成できません。transport input ssh で Telnet を締め出します。

Packet Tracerでの動作確認

PC の Desktop → Telnet / SSH Client から SSH を選び、ユーザー名 admin で R1 の IP に接続します。続けて Telnet でも接続を試み、拒否されることまで確認すると設定の意味が体感できます。

Port Security

ポートセキュリティ: 勝手な機器の接続を防ぐ

オフィスの床下や会議室の LAN ポートには、誰でも物理的に機器をつなげてしまいます。ポートセキュリティは、Switch のポートごとに学習するMACアドレスの数や値を制限し、想定外の機器がつながったときに自動で対処する機能です。

設定例: Fa0/1 をPC-A専用にする

SW1(config)# interface fastEthernet0/1 SW1(config-if)# switchport mode access SW1(config-if)# switchport port-security SW1(config-if)# switchport port-security maximum 1 SW1(config-if)# switchport port-security mac-address sticky SW1(config-if)# switchport port-security violation shutdown

sticky は「最初につながった機器の MAC アドレスを自動で覚えて固定する」設定です。ポートセキュリティは access モードのポートにしか設定できない点に注意してください。

違反時の3つの動作

  • shutdown(初期値): ポートを err-disabled 状態にして完全に停止する
  • restrict: 違反フレームを破棄し、ログと違反カウントを記録する
  • protect: 違反フレームを黙って破棄する(記録なし)

状態確認と、err-disabled になったポートの復旧はこう行います。

SW1# show port-security interface fastEthernet0/1 SW1(config)# interface fastEthernet0/1 SW1(config-if)# shutdown SW1(config-if)# no shutdown

実験のすすめ: わざと違反させる

PC-A で sticky 学習させたあと、ケーブルを別の PC につなぎ替えてトラフィックを流すと、ポートが err-disabled になりリンクランプが赤になります。show port-security の違反カウントとあわせて観察してください。「復旧には shutdown → no shutdown が必要」という点は実務でも頻出です。

Operations

運用の土台: SyslogとNTP

障害対応の第一歩は「いつ・何が起きたか」を知ることです。ログを1か所に集める Syslog と、全機器の時刻を揃える NTP はセットで設定します。時刻がずれていると、複数機器のログを突き合わせて原因を追うことができません。

Syslog: ログをサーバーに集める

R1(config)# logging 192.168.10.100 R1# show logging

Packet Tracer では Server を配置し、Services タブの SYSLOG をオンにすると受信ログを一覧できます。インターフェースの up/down などのイベントがサーバーに届く様子を確認しましょう。

NTP: 時刻を揃える

R1(config)# ntp server 192.168.10.100 R1# show ntp status R1# show clock

Server の Services タブで NTP を有効にし、Router / Switch をそのサーバーに同期させます。show ntp status に synchronized と表示されれば成功です。

やること コマンド / 操作 実務での意味
設定のバックアップ copy running-config startup-config、外部保存は copy running-config tftp: 機器故障時に同じ設定を即座に復元できるよう、変更のたびに保存・退避します。
隣接機器の確認 show cdp neighbors 配線図が古くても、実際に「どのポートが何とつながっているか」を機器自身から取得できます。
ログの確認 show logging 障害の時刻と直前のイベントを特定する、切り分けの出発点です。
Mini Lab

この単元のラボ構成

基礎ラボの構成(PC・Switch・Router)に Server を1台追加するだけで、この単元の内容をすべて練習できます。

PC-A管理端末192.168.10.10 /24
SW1Catalyst 2960Fa0/1: ポートセキュリティ
R1ISR 1941SSH管理の対象
ServerSYSLOG / NTP192.168.10.100 /24

このラボでやること

  • R1 と SW1 を SSH 接続のみに制限し、PC-A から SSH でログインする
  • SW1 の Fa0/1 にポートセキュリティ(sticky・shutdown)を設定する
  • 別の PC につなぎ替えて err-disabled を発生させ、復旧手順を実施する
  • R1・SW1 のログを Server(192.168.10.100)の SYSLOG に集約する
  • NTP 同期後、ログのタイムスタンプが揃っていることを確認する

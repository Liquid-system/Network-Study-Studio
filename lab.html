Basic Lab / Packet Tracer

最初に固めたい、Packet Tracerの基礎ラボ。

ここでは PC-A・SW1・R1 を使って、小さなネットワークを実際に組みます。手順どおりに疎通させるだけでなく、「なぜその設定が必要なのか」まで説明できるようになることがゴールです。

Topology

今回の構成

PC-AEnd Device192.168.10.10 /24
GW 192.168.10.1
SW1Catalyst 2960Access VLAN 10
R1ISR 1941G0/1: 192.168.10.1 /24

ゴール

  • PC-A から R1 へ Ping が通る
  • Router のインターフェースが up になる
  • Switch 側のポートが VLAN 10 に所属する

使うアドレス

  • PC-A: 192.168.10.10 / 255.255.255.0
  • Default Gateway: 192.168.10.1
  • R1 G0/1: 192.168.10.1 / 255.255.255.0

Packet Tracer で選ぶ Router は 1941

このラボのコマンド例は、GigabitEthernet ポート(G0/0・G0/1)を持つ 1941(2911 でも可)を選んだ前提です。1841 のような FastEthernet 搭載機種を選んだ場合は、interface gigabitEthernet0/1 を interface fastEthernet0/1 のようにポート名を読み替えてください。実際のポート名は show ip interface brief で確認できます。詳しくはPacket Tracer の使い方の「装置の選び方とポート名の関係」を参照してください。

Configuration Steps

Step by Stepで設定する

STEP 1

機器を配置して接続する

PC、2960 Switch、Router を1台ずつ置き、PC と Switch、Switch と Router の順にケーブルで接続します。

STEP 2

PC-AにIPを設定する

PC-A → Desktop → IP Configuration の順に開き、次の値を入力します。

IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1
STEP 3

SW1でVLAN 10とアクセスポートを設定する

VLAN 10 を作成し、PC-A がつながるポートを VLAN 10 の access ポートにします。

Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name USERS Switch(config-vlan)# exit Switch(config)# interface fastEthernet0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# description PC-A Switch(config-if)# end Switch# copy running-config startup-config
STEP 4

R1のインターフェースを設定する

Switch につながるポートに IP アドレスを設定し、R1(config-if)# no shutdown で有効化します。

Router> enable Router# configure terminal Router(config)# hostname R1 R1(config)# interface gigabitEthernet0/1 R1(config-if)# description Link-to-SW1 R1(config-if)# ip address 192.168.10.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# end R1# copy running-config startup-config
STEP 5

確認コマンドとPingを実行する

Router 側で状態を確認したあと、PC-A の Command Prompt から 192.168.10.1 へ Ping を打ちます。応答が返ってくれば成功です。

R1# show ip interface brief R1# show running-config
Why It Works

このラボで本当に理解したい3つのこと

設定を丸写しして終わりにしないために、特に大事な意味を3つに絞って整理します。

PCのGatewayは出口

PC は、自分と別のネットワークへ通信するとき、どこへ送ればよいかをデフォルトゲートウェイで判断します。今回はその出口が 192.168.10.1(R1)です。

Switchは同一LANの交通整理役

Switch は主に L2 で動きます。今回ポートを VLAN 10 に所属させるのは、「この PC はこの LAN の一員だ」と明示するためです。

Routerは no shutdown で動き出す

Router のポートは、IP アドレスを入れただけでは有効になりません。R1(config-if)# no shutdown を入力して初めて、通信できる状態になります。

Verification

設定後に確認したいポイント

Router側の確認

R1# show ip interface brief R1# show running-config

g0/1 が up/up になっているか、IP アドレスが正しく入っているかを確認します。

Switch側の確認

Switch# show vlan brief

PC-A がつながるポートが VLAN 10 に入っているかを確認します。

If It Fails

Pingが通らないときの確認順

1. PC設定

IP アドレス、サブネットマスク、デフォルトゲートウェイに入力ミスがないかを確認します。

2. Routerポート状態

R1# show ip interface brief で、対象のインターフェースが up/up になっているかを確認します。

3. Switchポート

PC-A を接続したポートが VLAN 10 に割り当てられているかを確認します。

ひとつ上の視点で見るポイント

PC から Router へ Ping が通ったということは、L1(ケーブル)・L2(Switch)・L3(IP)の最小条件がすべて揃ったということです。単に「成功した」で終わらせず、何が成立したのかを自分の言葉で説明できると、理解が一段深まります。

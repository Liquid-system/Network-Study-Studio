Basic Lab / Packet Tracer

最初に固めたい、Packet Tracerの基礎ラボ。

ここでは Packet Tracer 上で PC-A、SW1、R1 を使って小さなネットワークを組みます。目的は「手順を追えば疎通まで行ける」を超えて、「なぜその設定が必要か」まで説明できるようになることです。

Topology

今回の構成

PC-AEnd Device
192.168.10.10 /24
GW 192.168.10.1
SW1Catalyst 2960
Access VLAN 10
R1Router
G0/1: 192.168.10.1 /24

ゴール

  • PC-AからR1へPingが通る
  • Routerのインターフェースが up になる
  • Switch側のポートが VLAN 10 に所属する

使うアドレス

  • PC-A: 192.168.10.10 / 255.255.255.0
  • Default Gateway: 192.168.10.1
  • R1 G0/1: 192.168.10.1 / 255.255.255.0
Configuration Steps

Step by Stepで設定する

STEP 1

機器を配置して接続する

PC、2960 Switch、Routerを1台ずつ置きます。PC-Switch、Switch-Routerの順にケーブル接続します。

STEP 2

PC-AにIPを設定する

PC-A → Desktop → IP Configuration で以下を入力します。

IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1
STEP 3

SW1でVLAN 10とアクセスポートを設定する

PC-A がつながるポートを VLAN 10 の access ポートにします。

Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name USERS Switch(config-vlan)# exit Switch(config)# interface fastEthernet0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# description PC-A Switch(config-if)# end Switch# copy running-config startup-config
STEP 4

R1のインターフェースを設定する

SwitchにつながるポートへIPを入れ、R1(config-if)# no shutdown で有効化します。

Router> enable Router# configure terminal Router(config)# hostname R1 R1(config)# interface gigabitEthernet0/1 R1(config-if)# description Link-to-SW1 R1(config-if)# ip address 192.168.10.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# end R1# copy running-config startup-config
STEP 5

確認コマンドとPingを実行する

Routerで状態を見たあと、PC-Aから 192.168.10.1 にPingします。

R1# show ip interface brief R1# show running-config
Why It Works

このラボで本当に理解したい3つのこと

設定を丸ごと写して終わりにしないために、特に大事な意味を3点に絞って整理します。

PCのGatewayは出口

PCは、自分と同じネットワーク外に行くとき、どこへ送ればよいかをデフォルトゲートウェイで判断します。今回は 192.168.10.1 がその役目です。

Switchは同一LANの交通整理役

Switchは主にL2で動きます。今回VLAN 10 にポートを所属させるのは、「このPCはこのLANにいる」と明示する意味があります。

Routerは no shutdown で動き出す

Routerのポートは、IPを入れただけでは使えないことがあります。R1(config-if)# no shutdown によって初めて通信可能な状態になります。

Verification

設定後に確認したいポイント

Router側の確認

R1# show ip interface brief R1# show running-config

g0/1 が up/up になっているか、IPアドレスが正しいかを見ます。

Switch側の確認

Switch# show vlan brief

PC-A がつながるポートが VLAN 10 に入っているかを確認します。

If It Fails

Pingが通らないときの確認順

1. PC設定

IP、マスク、デフォルトゲートウェイに入力ミスがないか確認します。

2. Routerポート状態

R1# show ip interface brief で対象インターフェースが up/up か確認します。

3. Switchポート

接続したポートが VLAN 10 に割り当てられているか確認します。

ひとつ上の視点で見るポイント

PCからRouterへPingが通るということは、L1/L2/L3の最小条件が揃ったという意味です。単に「成功した」ではなく、何が成立したのかを言葉にできると理解が深まります。

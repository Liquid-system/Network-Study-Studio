Troubleshooting / Packet Tracer

通信できないときに、何から見るかを迷わない。

ネットワーク学習では、設定できることと同じくらい「崩れたときに戻せること」が大切です。ここでは、Packet Tracerで再現しやすい失敗例を使って、切り分けの順番を固めます。

Unit Topology

この単元で切り分け対象にする構成

トラブルシュートでは、どの単元でも応用できる代表構成を使って考えます。PC-A から別ネットワークの PC-B へ通信する流れを前提に、どこが崩れるとどんな症状になるかを見ていきます。

この図で見たいこと

物理接続、VLAN、IP、Gateway、経路のどこかが崩れると通信できなくなります。切り分けは、上から順に原因を減らしていく作業です。

代表的な確認対象

  • PC-A の Gateway が 192.168.10.1 になっているか
  • SW1 のポートが正しい VLAN に所属しているか
  • R1 のインターフェースが up/up か
Golden Order

まずはこの順番で確認する

1

物理接続を見る

ケーブル接続、機器の電源、ポートのリンク状態を見ます。L1が崩れていると、その上をいくら見ても前に進みません。

2

L2の状態を見る

VLAN、access / trunk、MAC学習状況を確認します。特にVLANの割り当て違いは、Packet Tracerの学習初期で頻出です。

3

L3の設定を見る

IPアドレス、サブネットマスク、デフォルトゲートウェイ、インターフェースの up/down を確認します。

4

経路を見る

別セグメント通信なら Router# show ip route でルーティング情報を確認します。静的ルートの抜けや次ホップ間違いはここで見つかります。

5

最後に疎通試験をする

途中経過を確認した上で、Ping や Simulation を使ってどこまで届いているかを見ます。最初からPingだけ打つと原因がぼやけやすいです。

Where to Look

Packet Tracerで見る場所

PCの Desktop / Command Prompt

IP設定画面でGatewayを見直し、Command Promptで ping や ipconfig 的な確認を行います。

Switch / Router のCLI

Router# show ip interface brief、Switch# show vlan brief、Router# show ip route が基本の確認セットです。

Simulationモード

どの機器までパケットが届いたかを視覚的に追えます。設定ミスの場所を狭めるのに非常に役立ちます。

Common Scenarios

よくある失敗パターンと見抜き方

症状: 同一LANなのにPingが通らない

  • まず物理接続とポート状態を見る
  • 次にPCのIPとマスクを見る
  • Switch側のVLAN割り当てが意図通りか確認する

症状: Routerの向こう側だけ届かない

  • PCのGatewayが正しいか確認する
  • Routerのインターフェースが up/up か見る
  • Router# show ip route で遠隔ネットワークの経路を見る

症状: VLAN間通信だけ失敗する

  • trunkポート設定を確認する
  • Routerサブインターフェースと VLAN 番号の対応を見る
  • 各PCのGatewayが VLAN ごとのRouter IPになっているか確認する
Command Checklist

迷ったらここから打つ確認コマンド

Switch

Switch# show interfaces status Switch# show vlan brief Switch# show mac address-table

Router

Router# show ip interface brief Router# show running-config Router# show ip route

考え方のコツ

「設定を全部読む」のではなく、「この症状なら次に何を見るべきか」を決めてからコマンドを打つと、切り分けが早くなります。これはPacket Tracerでも実機でも同じです。

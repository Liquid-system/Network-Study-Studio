ダーク ライト

IPv6 は「別物の難しい世界」ではなく、IPv4 で学んだ考え方の置き換えです。IPv4 のどの仕組みに対応するかを意識しながら読むと、一気に理解しやすくなります。

IPv6 Basics / Addressing & Routing

IPv4の「次」を、恐れずに読めるようになる。

IPv4 アドレスは約43億個で、すでに新規割り当ては枯渇しています。その解決策が128ビットの IPv6 です。桁が長く見た目に圧倒されがちですが、読み方のルールは3つだけ。この単元で「読める・書ける・設定できる」を一度に身につけます。

Address Format

IPv6アドレスの読み方は3つのルールで決まる

IPv6 アドレスは128ビットを16進数で表し、16ビットずつコロンで8ブロックに区切ります。例: 2001:0db8:0000:0000:0000:0000:0000:0001。長く見えますが、省略ルールを使うと 2001:db8::1 まで短くなります。

ルール1: 先頭のゼロは省略

各ブロックの先頭の連続するゼロは省略できます。0db8 は db8、0000 は 0 と書けます。

ルール2: 連続するゼロは «::»

ゼロだけのブロックが連続する部分は、アドレス内で1回だけ :: に置き換えられます。2回使うと元の長さが復元できなくなるため禁止です。

ルール3: プレフィックスは /64 が基本

サブネットマスクの代わりに /64 のようなプレフィックス長で書きます。LAN では前半64ビットがネットワーク部、後半64ビットがホスト部というのが標準です。

省略の練習

2001:0db8:0000:0000:0000:0000:0000:0001 → 先頭ゼロを取ると 2001:db8:0:0:0:0:0:1 → 連続するゼロを :: にまとめると 2001:db8::1。この変換が暗算でできれば、IPv6 の見た目はもう怖くありません。

Address Types

アドレスの種類と、IPv4との対応関係

IPv6 には用途別のアドレスが複数あり、1つのインターフェースが同時に複数のアドレスを持つのが普通です。特に「リンクローカルアドレスは必ず自動で付く」ことを覚えてください。

種類 範囲 役割
グローバルユニキャスト 2000::/3 インターネットで一意に使えるアドレス。IPv4 のグローバルIPに相当します。ドキュメントや学習では 2001:db8::/32 を使う決まりです。
リンクローカル fe80::/10 同一リンク内だけで有効なアドレスで、IPv6 が有効なインターフェースに自動で必ず付与されます。ルーティングプロトコルのネイバー通信などに使われます。
ユニークローカル fc00::/7 組織内だけで使うアドレス。IPv4 のプライベートIP(192.168.x.x など)に相当します。
マルチキャスト ff00::/8 特定グループへの一斉送信。IPv6 にはブロードキャストが存在せず、その役割はマルチキャストが担います。

IPv4の仕組みはこう置き換わる

  • ARP → NDP(近隣探索プロトコル)がMACアドレスを解決
  • ブロードキャスト → マルチキャストに統合
  • DHCP → SLAAC(Router の広告から自動生成)または DHCPv6
  • NAT → 原則不要(アドレスが豊富なためエンドツーエンドで通信)

SLAACとは

SLAAC(ステートレスアドレス自動設定)では、Router が広告するプレフィックス(例: 2001:db8:10::/64)を受け取った PC が、ホスト部を自分で生成してアドレスを完成させます。DHCP サーバーがなくてもアドレスが決まるのが IPv6 らしい仕組みです。Packet Tracer では PC の IP Configuration で「Automatic」を選ぶと体験できます。

Configuration

RouterにIPv6を設定する

IPv4 と大きく違う注意点が1つ: Cisco Router は初期状態で IPv6 のルーティングが無効です。インターフェースにアドレスを設定する前に、まず ipv6 unicast-routing を有効にします。

PC-A2001:db8:10::10 /64GW 2001:db8:10::1
R1ISR 1941G0/0: 2001:db8:10::1
G0/1: 2001:db8:20::1
PC-B2001:db8:20::10 /64GW 2001:db8:20::1

R1 の設定

R1(config)# ipv6 unicast-routing R1(config)# interface gigabitEthernet0/0 R1(config-if)# ipv6 address 2001:db8:10::1/64 R1(config-if)# no shutdown R1(config-if)# exit R1(config)# interface gigabitEthernet0/1 R1(config-if)# ipv6 address 2001:db8:20::1/64 R1(config-if)# no shutdown

PC の設定と疎通確認

PC の Desktop → IP Configuration で IPv6 欄に入力します。

PC-A IPv6 Address: 2001:db8:10::10 / 64 IPv6 Gateway: 2001:db8:10::1 PC-A> ping 2001:db8:20::10

確認コマンドは IPv4 の show ip ... を show ipv6 ... に読み替えるだけです。

R1# show ipv6 interface brief R1# show ipv6 route

つまずきポイント: ping が通らないとき

まず ipv6 unicast-routing を入れ忘れていないかを疑ってください。インターフェースにアドレスが付いていても、これが無効だと R1 はパケットを転送しません。次に show ipv6 interface brief で、各インターフェースが up/up でグローバルアドレスとリンクローカルアドレス(FE80::...)の両方を持っているかを確認します。

Routing

IPv6でも、静的ルートとOSPFの考え方はそのまま

静的ルートとデフォルトルート

書式は IPv4 と同じ発想で、宛先プレフィックスとネクストホップを指定します。

R1(config)# ipv6 route 2001:db8:30::/64 2001:db8:20::2 R1(config)# ipv6 route ::/0 2001:db8:20::2

::/0 が IPv4 の 0.0.0.0 0.0.0.0(デフォルトルート)に相当します。

OSPFv3

IPv6 用の OSPF は OSPFv3 と呼ばれます。IPv4 の OSPF と違い network コマンドではなく、インターフェースに直接エリアを指定するのが特徴です。

R1(config)# ipv6 router ospf 1 R1(config-rtr)# router-id 1.1.1.1 R1(config-rtr)# exit R1(config)# interface gigabitEthernet0/0 R1(config-if)# ipv6 ospf 1 area 0

Dynamic Routing / OSPF / Single Area

経路を手で書く世界から、自動で学ぶ世界へ。

中級ラボでは、Router に静的ルートを1本ずつ手で入力しました。この単元では Router を3台に増やし、静的ルートの限界を体感したうえで、OSPF に経路を自動で学習させます。動的ルーティングの第一歩として、シングルエリア(Area 0)だけに絞って学びます。

Why Dynamic Routing

なぜ静的ルートだけでは足りないのか

Router が2台までなら静的ルートで十分戦えます。しかし3台以上になると、書くべきルートの本数が急に増え、構成変更のたびに全 Router を手作業で直すことになります。

静的ルートの場合

  • 今回の3台構成では、合計6本の静的ルートを手で書く必要があります
  • ネットワークを1つ追加するたびに、離れた Router 全部に経路を追記します
  • 経路が壊れても、自動では復旧しません

OSPFの場合

  • 各 Router に「自分が持っているネットワーク」を宣言するだけで、残りは自動で交換されます
  • Router 同士が隣接関係(ネイバー)を結び、経路情報を常に同期します
  • リンクが切れると、経路を自動で計算し直します
Unit Topology

この単元で構築する構成

Router 3台を一直線につなぎ、両端の LAN に PC を1台ずつ置きます。Router 間のリンクには、2台しか入らない小さなネットワーク(/30)を使う点も、この単元の新しい学習ポイントです。

GW 192.168.1.1
R1ISR 1941G0/0: 192.168.1.1
G0/1: 10.0.12.1 /30
R2ISR 1941G0/0: 10.0.12.2 /30
G0/1: 10.0.23.1 /30
R3ISR 1941G0/0: 10.0.23.2 /30
G0/1: 192.168.3.1 /24
PC-BEnd Device192.168.3.10 /24

アドレス表

  • PC-A の LAN: 192.168.1.0 /24
  • R1 と R2 の間: 10.0.12.0 /30(R1 が .1、R2 が .2)
  • R2 と R3 の間: 10.0.23.0 /30(R2 が .1、R3 が .2)
  • PC-B の LAN: 192.168.3.0 /24

/30 を使う理由

Router 同士をつなぐリンクには機器が2台しかいません。/30(サブネットマスク 255.255.255.252)は使える IP がちょうど2個のネットワークなので、Router 間リンクの定番として使われます。/24 との違いを意識しながら設定してみてください。

Packet Tracer で選ぶ Router は 1941

R1〜R3 の3台とも、GigabitEthernet ポート(G0/0・G0/1)を持つ 1941(2911 でも可)を選んでください。1841 のような FastEthernet 搭載機種を選んだ場合は、interface gigabitEthernet0/0 を interface fastEthernet0/0 のようにポート名を読み替える必要があります。実際のポート名は show ip interface brief で確認できます。詳しくはPacket Tracer の使い方の「装置の選び方とポート名の関係」を参照してください。

Step by Step

OSPFを動かすまでの手順

STEP 1

機器を配置して IP を設定する

Router 3台と PC 2台を配置し、まずは各インターフェースに IP を設定します。ここは基礎ラボ・中級ラボの復習です。R1 の例を示すので、R2 と R3 もアドレス表どおりに設定してください。

R1(config)# interface gigabitEthernet0/0 R1(config-if)# ip address 192.168.1.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# exit R1(config)# interface gigabitEthernet0/1 R1(config-if)# ip address 10.0.12.1 255.255.255.252 R1(config-if)# no shutdown R1(config-if)# exit
STEP 2

隣どうしの疎通を確認する

OSPF を設定する前に、R1 から 10.0.12.2(R2)へ、R2 から 10.0.23.2(R3)へ Ping を打ちます。隣接リンクが通らない状態で OSPF を入れても、ネイバーは絶対に張れません。先に土台を固めるのがコツです。

STEP 3

R1 に OSPF を設定する

network コマンドで「OSPF に参加させるネットワーク」を宣言します。ここで使うワイルドカードマスクは、ACL 入門にも登場した「サブネットマスクの 0 と 255 を反転させたもの」です(/24 なら 0.0.0.255、/30 なら 0.0.0.3)。

R1(config)# router ospf 1 R1(config-router)# router-id 1.1.1.1 R1(config-router)# network 192.168.1.0 0.0.0.255 area 0 R1(config-router)# network 10.0.12.0 0.0.0.3 area 0 R1(config-router)# exit
STEP 4

R2 と R3 にも OSPF を設定する

各 Router は「自分に直接つながっているネットワーク」だけを宣言します。離れたネットワークを書く必要がないのが、静的ルートとの大きな違いです。

R2(config)# router ospf 1 R2(config-router)# router-id 2.2.2.2 R2(config-router)# network 10.0.12.0 0.0.0.3 area 0 R2(config-router)# network 10.0.23.0 0.0.0.3 area 0 R2(config-router)# exit R3(config)# router ospf 1 R3(config-router)# router-id 3.3.3.3 R3(config-router)# network 10.0.23.0 0.0.0.3 area 0 R3(config-router)# network 192.168.3.0 0.0.0.255 area 0 R3(config-router)# exit
STEP 5

ネイバーと経路を確認する

設定直後にコンソールへ表示される Full adjacency のメッセージが、ネイバー確立のサインです。コマンドでも状態を確かめます。

R1# show ip ospf neighbor R1# show ip route ospf R1# copy running-config startup-config

show ip ospf neighbor で State が FULL になっていれば、経路交換は完了しています。show ip route では、自分で書いていない 192.168.3.0/24 が O(OSPF)付きで載っているはずです。

STEP 6

疎通確認と「自動復旧」の観察

PC-A から PC-B へ Ping が通ることを確認したら、仕上げに OSPF らしさを体験しましょう。R2 の G0/1 を shutdown してから show ip route を見ると、失われた経路が経路表から自動で消えます。no shutdown で戻すと、数秒で経路が復活します。静的ルートでは起きない動きです。

Why It Works

OSPFの中で起きていること

1. ネイバーを見つける

OSPF を有効にした Router は、リンク上に Hello パケットを定期的に送り、同じ Area の相手を見つけて隣接関係を結びます。

2. 地図を共有する

各 Router は「自分がどのネットワークにつながっているか」という情報(LSA)を交換し、全員が同じネットワーク地図(LSDB)を持ちます。

3. 最短経路を計算する

共有した地図をもとに、各 Router が自分視点での最短経路を計算し、その結果だけを経路表に載せます。リンク断が起きると再計算します。

If It Fails

うまくいかないときの確認ポイント

ネイバーが張れない

まず R1# show ip ospf neighbor に相手が出てくるかを見ます。出てこない場合は、リンクの up/down と、network コマンドの範囲にそのインターフェースが含まれているかを確認します。

ワイルドカードの間違い

network 10.0.12.0 0.0.0.3 と書くべきところを 255.255.255.252 と書いてしまうのが定番のミスです。ワイルドカードマスクはサブネットマスクの反転だと覚えてください。

area 番号の不一致

今回はすべて area 0 に統一します。片方だけ area 1 になっていると、同じリンク上にいてもネイバーになれません。R1# show running-config で見直します。

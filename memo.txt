新章「ACL入門」の内容

PC 2台 + Switch + Router + Web Server の小さなトポロジで、「PC-A だけ止まり PC-B は通る」という選別を目で確かめられる構成にしました。

- 2つのタイプを両方体験: 番号で指定する標準 ACL(access-list 10 deny host ...)と、名前で指定する拡張 ACL(ip access-list extended BLOCK-A-WEB)を1つずつ作ります。番号の範囲(1〜99 / 100〜199)、名前付きなら行単位の削除・挿入(no 10)ができる利点も解説しています。
- コマンドの意味を1つずつ解説: 各 STEP で deny host と ワイルドカードマスクの関係、permit any がないと暗黙の deny で全滅する理由、ip access-group の意味を、コード例の直後で説明しています。
- 効果を実感する仕掛け: 「作っただけでは何も起きない」→適用して止まる瞬間の Destination host unreachable(タイムアウトとの違いも説明)→ show access-lists のヒット数が増える様子、という流れです。拡張 ACL では「Ping は通るのに Web だけ止まる」を体験します。

in / out の重点解説

専用セクションを設け、さらにハンズオンの中心に据えました。

- 「主人公は Router」という判定原則と、Ping 1往復で通過する4つの検査ポイント(行き G0/0 in → G0/1 out、帰り G0/1 in → G0/0 out)を図解
- STEP 4 で「わざと間違える実験」: 初学者がやりがちな「PC-A は LAN 側だから G0/0 の out」を実際に試すと Ping が通ってしまうことを確認し、帰りパケットの送信元は Server だから deny に一致しない、という理由まで追いかけます。STEP 5 で in に付け替えて結果が反転するのを確認し、理解度チェックの Q1 でも同じ観点を出題しています。

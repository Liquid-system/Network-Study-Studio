L3 Switch / SVI / Inter-VLAN Routing

SVI と L3 Switch を、最小構成から理解する。

この単元では、Router を使わずに Multilayer Switch だけで VLAN 間通信を成立させます。SVI の意味、`ip routing` の役割、L2 Switch との違いを step by step で整理します。

Unit Topology

この単元で構築する構成

この単元では Multilayer Switch 1台と PC 2台だけで VLAN 間通信を成立させます。SVI が各 VLAN のゲートウェイになり、MLS1(config)# ip routing で L3 機能を有効にするのが要点です。

PC-AEnd DeviceVLAN 10
192.168.10.10 /24
MLS1Catalyst 3560SVI VLAN 10: 192.168.10.1
SVI VLAN 20: 192.168.20.1
VLAN間ルーティングMLS1(config)# ip routing
PC-BEnd DeviceVLAN 20
192.168.20.10 /24

L2 Switch との違い

  • L2 Switch は VLAN を分けられても、通常は VLAN 間をルーティングしません
  • L3 Switch は SVI に IP を持たせ、VLAN 間を内部でルーティングできます
  • Router-on-a-Stick より、構成がシンプルになる場面があります

今回の役割分担

  • PC-A は VLAN 10 に所属し、192.168.10.1 を Gateway に使います
  • PC-B は VLAN 20 に所属し、192.168.20.1 を Gateway に使います
  • MLS1 の SVI が、それぞれの VLAN の出口になります

この構成で学べること

  • VLAN ごとにデフォルトゲートウェイを用意する意味
  • SVI が VLAN の出口として振る舞う仕組み
  • MLS1(config)# ip routing が必要な理由
Step by Step

最小構成から設定する手順

STEP 1

PC 2台と Multilayer Switch 1台を置く

Packet Tracer では 3560 系スイッチを使うと、SVI と ip routing を学びやすいです。PC-A を Fa0/1、PC-B を Fa0/2 に接続します。

STEP 2

PC-A と PC-B に IP を設定する

PC-A IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1 PC-B IP Address: 192.168.20.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.20.1
STEP 3

VLAN と access ポートを作る

まずは PC-A を VLAN 10、PC-B を VLAN 20 に所属させます。

Switch> enable Switch# configure terminal Switch(config)# hostname MLS1 MLS1(config)# vlan 10 MLS1(config-vlan)# name USERS-10 MLS1(config-vlan)# exit MLS1(config)# vlan 20 MLS1(config-vlan)# name USERS-20 MLS1(config-vlan)# exit MLS1(config)# interface fastEthernet0/1 MLS1(config-if)# switchport mode access MLS1(config-if)# switchport access vlan 10 MLS1(config-if)# description PC-A MLS1(config-if)# exit MLS1(config)# interface fastEthernet0/2 MLS1(config-if)# switchport mode access MLS1(config-if)# switchport access vlan 20 MLS1(config-if)# description PC-B MLS1(config-if)# exit
STEP 4

SVI に IP アドレスを設定する

VLAN 10 と VLAN 20 の論理インターフェースに IP を付けることで、各 VLAN の Gateway を作ります。

MLS1(config)# interface vlan 10 MLS1(config-if)# ip address 192.168.10.1 255.255.255.0 MLS1(config-if)# no shutdown MLS1(config-if)# exit MLS1(config)# interface vlan 20 MLS1(config-if)# ip address 192.168.20.1 255.255.255.0 MLS1(config-if)# no shutdown MLS1(config-if)# exit
STEP 5

L3 Switch としてルーティングを有効にする

SVI を作っただけでは、VLAN 間をルーティングしない機種や設定があります。MLS1(config)# ip routing で L3 機能を有効にします。

MLS1(config)# ip routing MLS1(config)# end MLS1# copy running-config startup-config
STEP 6

確認する

PC-A から PC-B へ Ping を試し、MLS1 側でも SVI とルーティング状態を確認します。

MLS1# show ip interface brief MLS1# show vlan brief MLS1# show ip route
Why It Works

Router-on-a-Stick と何が違うか

Router-on-a-Stick

Router のサブインターフェースを使い、Switch との trunk リンクを通して VLAN 間通信を実現します。外部 Router を使うので構成要素が増えます。

L3 Switch / SVI

Switch 自身が各 VLAN の Gateway を持ち、そのまま内部でルーティングします。最小構成では Router が不要なので、SVI の考え方に集中しやすいです。

If It Fails

うまくいかないときの確認ポイント

VLAN番号

PC-A のポートが VLAN 10、PC-B のポートが VLAN 20 に本当に入っているかを MLS1# show vlan brief で見ます。

SVIの状態

MLS1# show ip interface brief で VLAN 10 / VLAN 20 の SVI が up になっているか確認します。

ip routing

MLS1# show running-config で ip routing が入っているか確認します。SVI があっても、これが抜けると VLAN 間通信ができないことがあります。

Cisco IOS / Command Flow

初心者が最初に覚えるCiscoコマンドの流れ。

大切なのは、いきなり設定を書き込むことではなく「まず確認し、次に設定し、最後に保存する」という流れです。実際の現場と同じこの順番を、Packet Tracer で練習します。

Unit Topology

この単元で意識する構成

Cisco CLI の基本は、PC・L2 Switch・Router の最小構成で学ぶのが一番の近道です。どの機器にどのコマンドを入れるのかを、まず構成図と IP アドレスで確認しておきましょう。

PC-AEnd Device192.168.10.10 /24
GW 192.168.10.1
SW1Catalyst 2960Access Port
R1ISR 1941G0/1: 192.168.10.1 /24

CLIを主に触る装置

このページでは Router の CLI を中心に学びますが、Switch 側でも Switch# show vlan brief のような基本の確認コマンドを使います。

IP値の前提

Router の LAN 側インターフェースに 192.168.10.1 / 255.255.255.0 を設定する想定です。この IP が、そのまま PC-A のデフォルトゲートウェイになります。

Packet Tracer で選ぶ Router は 1941

この単元のコマンド例は、GigabitEthernet ポート(G0/0・G0/1)を持つ 1941(2911 でも可)を選んだ前提です。1841 のような FastEthernet 搭載機種を選んだ場合は、interface gigabitEthernet0/1 を interface fastEthernet0/1 のようにポート名を読み替えてください。実際のポート名は show ip interface brief で確認できます。詳しくはPacket Tracer の使い方の「装置の選び方とポート名の関係」を参照してください。

Step by Step

最初の基本操作

01

特権モードへ移る

Router> enable と入力すると、プロンプトが Router# に変わります。ここから使える確認コマンドの幅が広がります。

Router> enable
02

現状を確認する

設定を触る前に、まず現在の状態を確認します。インターフェースの状態や、すでに入っている設定をここで把握します。

Router# show ip interface brief Router# show running-config
03

設定モードに入る

Router# configure terminal で設定モードに入ります。機器名(hostname)の変更やインターフェース設定は、ここから始めます。

Router# configure terminal Router(config)# hostname R1
04

インターフェースにIPを設定する

対象のインターフェースに移動して IP アドレスとサブネットマスクを設定し、最後に R1(config-if)# no shutdown でポートを有効化します。

R1(config)# interface gigabitEthernet0/1 R1(config-if)# ip address 192.168.10.1 255.255.255.0 R1(config-if)# no shutdown
05

保存する

設定が終わったら必ず保存します。保存しないと、再起動したときに設定が消えてしまいます。Packet Tracer のうちからこの癖をつけておくと、実機に移ったときに必ず役立ちます。

R1(config-if)# end R1# copy running-config startup-config
Cheat Sheet

最初に覚える代表コマンド

確認系

  • Router# show ip interface brief でポートの状態とIPを見る
  • Router# show running-config で現在の設定を見る
  • Switch# show vlan brief でVLANを見る
  • Router# show ip route で経路情報を見る

設定系

  • Router# configure terminal で設定を開始する
  • Router(config)# hostname Branch-R1 で機器名を設定する
  • Router(config)# interface g0/1 で対象ポートへ移動する
  • Router(config-if)# ip address ... と Router(config-if)# no shutdown でL3ポートを設定する

CLI学習のコツ

コマンドを1行ずつ丸暗記するより、「enable で特権モードへ → show で現状確認 → configure terminal で設定 → copy running-config startup-config で保存」という一連の流れを体で覚えるほうが、ずっと定着します。

Cisco IOS / Command Flow

初心者が最初に覚えるCiscoコマンドの流れ。

現場でも大事なのは、いきなり設定を書くことではなく「確認してから設定し、最後に保存する」流れです。Packet Tracerでも同じ順番で練習します。

CLIを主に触る装置

このページでは Router の CLI を中心に学びますが、Switch 側でも Switch# show vlan brief のような基本確認コマンドを使います。

IP値の前提

Router の LAN 側インターフェースには 192.168.10.1 / 255.255.255.0 を設定する想定です。これが PC-A のゲートウェイになります。

Step by Step

最初の基本操作

01

特権モードへ移る

Router> enable を入力して Router# のプロンプトへ移ります。ここから確認系コマンドの幅が広がります。

Router> enable
02

現状を確認する

まずは現在の状態を見ます。インターフェース状態や設定の有無をここで把握します。

Router# show ip interface brief Router# show running-config
03

設定モードに入る

Router# configure terminal で設定モードに入り、機器名やインターフェース設定を入れます。

Router# configure terminal Router(config)# hostname R1
04

インターフェースにIPを設定する

対象インターフェースを選んで、IPとマスクを設定し、最後に R1(config-if)# no shutdown を入れます。

R1(config)# interface gigabitEthernet0/1 R1(config-if)# ip address 192.168.10.1 255.255.255.0 R1(config-if)# no shutdown
05

保存する

終わったら必ず保存します。Packet Tracerでもこのクセをつけると実機に移ったときに役立ちます。

R1(config-if)# end R1# copy running-config startup-config
Cheat Sheet

最初に覚える代表コマンド

確認系

  • Router# show ip interface brief でポート状態とIPを見る
  • Router# show running-config で現在の設定を見る
  • Switch# show vlan brief でVLANを見る
  • Router# show ip route でルートを見る

設定系

  • Router# configure terminal で設定開始
  • Router(config)# hostname Branch-R1 で機器名設定
  • Router(config)# interface g0/1 で対象ポートへ移動
  • Router(config-if)# ip address ... と Router(config-if)# no shutdown でL3ポート設定

CLI学習のコツ

コマンドを1行ずつ丸暗記するより、「Router> enable → Router# show → Router# configure terminal → Router(config)# interface → Router# save」という順番を体で覚えるほうがずっと定着します。

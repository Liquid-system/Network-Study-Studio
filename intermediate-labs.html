Before You Start

取り組む前に確認したい前提

CLIの流れが頭にある

Router> enable、Router# configure terminal、Router(config)# interface、Router# show、保存の順番を把握していると進めやすいです。

分からなくなったら戻る

VLANやGatewayの意味が曖昧になったら、概念理解ページや基礎ラボページに戻って整理すると迷いにくくなります。

Lab 1

VLAN間通信: Router-on-a-Stick を組む

1台のSwitchの中で VLAN 10 と VLAN 20 を分け、Routerのサブインターフェースで相互通信させる構成です。

このラボで理解したいこと

  • VLANを分けると、同じSwitch上でもそのままでは通信できない
  • 複数VLANをRouterへ渡すには trunk が必要
  • Router側では VLAN ごとにサブインターフェースを作る

アドレス設計例

  • PC-A: 192.168.10.10 /24 Gateway 192.168.10.1
  • PC-B: 192.168.20.10 /24 Gateway 192.168.20.1
  • R1 G0/0.10: 192.168.10.1 /24
  • R1 G0/0.20: 192.168.20.1 /24
STEP 1

PC-A、PC-B、SW1、R1 を配置する

PC-A は VLAN 10 用、PC-B は VLAN 20 用として使います。両方とも同じSwitchに接続し、SwitchからRouterへ1本のリンクを張ります。

STEP 2

Switchに VLAN と access ポートを作る

PC-A のポートを VLAN 10、PC-B のポートを VLAN 20 に割り当てます。これでL2レベルでは別ネットワークとして分かれます。

Switch> enable Switch# configure terminal Switch(config)# vlan 10 Switch(config-vlan)# name USERS-10 Switch(config-vlan)# exit Switch(config)# vlan 20 Switch(config-vlan)# name USERS-20 Switch(config-vlan)# exit Switch(config)# interface fastEthernet0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 10 Switch(config-if)# description PC-A Switch(config-if)# exit Switch(config)# interface fastEthernet0/2 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 20 Switch(config-if)# description PC-B
STEP 3

Router接続ポートを trunk にする

複数のVLAN情報を1本のリンクで運ぶために trunk を使います。ここが access のままだと片方のVLANしか運べません。

Switch> enable Switch# configure terminal Switch(config)# interface fastEthernet0/24 Switch(config-if)# switchport mode trunk Switch(config-if)# description Uplink-to-R1 Switch(config-if)# end Switch# copy running-config startup-config
STEP 4

Routerにサブインターフェースを作る

1本の物理ポートの上に、VLAN 10 用と VLAN 20 用の論理インターフェースを作ります。encapsulation dot1Q が VLAN タグを扱う要点です。

Router> enable Router# configure terminal Router(config)# interface gigabitEthernet0/0 Router(config-if)# no shutdown Router(config-if)# exit Router(config)# interface gigabitEthernet0/0.10 Router(config-subif)# encapsulation dot1Q 10 Router(config-subif)# ip address 192.168.10.1 255.255.255.0 Router(config-subif)# exit Router(config)# interface gigabitEthernet0/0.20 Router(config-subif)# encapsulation dot1Q 20 Router(config-subif)# ip address 192.168.20.1 255.255.255.0 Router(config-subif)# end Router# copy running-config startup-config
STEP 5

PCのIPを設定して疎通確認する

PC-A から PC-B に Ping が通れば成功です。通らない場合は、Gateway、trunk、サブインターフェース番号を優先して確認します。

なぜこれで通信できるのか

VLANで分かれた端末同士は、そのままではL2の境界を越えられません。Routerが VLAN ごとのGatewayとなり、片方のVLANで受けたIPパケットを、もう片方のVLANへL3転送することで初めて相互通信できます。

Lab 2

静的ルーティング: 2台のRouterで別LAN同士をつなぐ

拠点Aと拠点Bを2台のRouterで接続し、互いのLANへ到達するための経路を静的ルートで教える構成です。

アドレス設計例

  • PC-A: 192.168.10.10 /24 Gateway 192.168.10.1
  • R1 G0/0: 192.168.10.1 /24
  • R1 G0/1: 10.0.0.1 /30
  • R2 G0/1: 10.0.0.2 /30
  • R2 G0/0: 192.168.20.1 /24
  • PC-B: 192.168.20.10 /24 Gateway 192.168.20.1

このラボで理解したいこと

  • Routerは、知らないネットワークへは勝手には送れない
  • 静的ルートは「どこへ送れば届くか」を明示する設定
  • 片方向だけでなく、戻り経路も必要
STEP 1

PC-A - R1 - R2 - PC-B の構成を置く

Switchを挟んでも良いですが、ルーティングの学習が主目的なので、ここでは Router 間接続の意味を意識することを優先します。

STEP 2

R1 と R2 のインターフェースにIPを設定する

各RouterにLAN側とRouter間リンク側のIPを入れ、すべて R1(config-if)# no shutdown / R2(config-if)# no shutdown で有効化します。

R1> enable R1# configure terminal R1(config)# interface gigabitEthernet0/0 R1(config-if)# ip address 192.168.10.1 255.255.255.0 R1(config-if)# no shutdown R1(config-if)# exit R1(config)# interface gigabitEthernet0/1 R1(config-if)# ip address 10.0.0.1 255.255.255.252 R1(config-if)# no shutdown R2> enable R2# configure terminal R2(config)# interface gigabitEthernet0/0 R2(config-if)# ip address 192.168.20.1 255.255.255.0 R2(config-if)# no shutdown R2(config-if)# exit R2(config)# interface gigabitEthernet0/1 R2(config-if)# ip address 10.0.0.2 255.255.255.252 R2(config-if)# no shutdown
STEP 3

PC-A と PC-B のIPを設定する

それぞれ自分のLAN側RouterをGatewayに設定します。ここが違うと、PCは別セグメントへ出ていけません。

STEP 4

静的ルートを入れる

R1 には 192.168.20.0/24 への行き先を、R2 には 192.168.10.0/24 への行き先を教えます。

R1(config)# ip route 192.168.20.0 255.255.255.0 10.0.0.2 R1(config)# end R1# copy running-config startup-config R2(config)# ip route 192.168.10.0 255.255.255.0 10.0.0.1 R2(config)# end R2# copy running-config startup-config
STEP 5

確認する

R1# show ip route で静的ルートが入っているか確認し、PC-A から PC-B へPingします。可能なら tracert でRouterを経由していることも観察します。

失敗しやすいポイント

片方のRouterだけに静的ルートを入れて満足してしまうケースが多いです。片道で届いても戻り経路がなければ通信は成立しません。また、/30 のマスク入力ミスも非常に起きやすいです。

Stretch Goals

余裕があれば試したい発展課題

VLAN 30 を追加する

3つ目の VLAN を足して、Router-on-a-Stick の仕組みが本当に理解できているか確認します。

静的ルートを一度消してみる

通信が止まることを確認してから戻すと、経路設定の意味がより鮮明になります。

Simulationで追跡する

Packet TracerのSimulationモードで、VLAN間やRouter間をパケットがどう通るかを可視化してみてください。

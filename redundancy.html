Redundancy / STP・EtherChannel・HSRP

機器が1台壊れても、通信を止めない。

実務のネットワークでは、Switch もケーブルも Router も「いつか壊れるもの」として設計します。この単元では、経路を二重化しつつループを防ぐ STP、リンクを束ねる EtherChannel、デフォルトゲートウェイを二重化する HSRP を学びます。

Why Redundancy

二重化すると、今度は「ループ」が問題になる

Switch 同士をケーブル2本でつなぐと、片方が切れても通信を継続できます。ところが L2 ネットワークでは、経路が輪(ループ)になるとブロードキャストフレームが永遠に回り続け、ネットワーク全体が麻痺します。これをブロードキャストストームと呼びます。

単一障害点(SPOF)をなくす

ケーブル1本・機器1台だけに依存した構成では、その1点が壊れた瞬間に全体が止まります。実務では「どこが壊れたら通信が止まるか」を常に意識し、重要な経路と機器を二重化します。

L2ループが危険な理由

IP パケットには TTL があり、ループしてもいずれ破棄されます。しかし L2 の Ethernet フレームには TTL に相当する仕組みがありません。ループ経路に入ったブロードキャストは減ることなく増殖し続けます。だから L2 の冗長化には STP が必須なのです。

Spanning Tree Protocol

STP: ループを自動でブロックする

STP(スパニングツリープロトコル)は、Switch 同士が BPDU というメッセージを交換し、ループになる経路の一部を自動的にブロック(通信停止状態)にする仕組みです。障害が起きると、ブロックしていたポートを開放して経路を切り替えます。

ルートブリッジの選出

STP はまず、ネットワーク内の Switch から1台の「ルートブリッジ」を選びます。ブリッジID(プライオリティ + MACアドレス)が最も小さい Switch が選ばれ、全経路はルートブリッジを起点に計算されます。

プライオリティの初期値はどの Switch も 32768 なので、何も設定しないと「MACアドレスが最も小さい(=たいてい最も古い)Switch」がルートになってしまいます。実務では中心の Switch を意図的にルートにします。

SW1(config)# spanning-tree vlan 1 priority 4096

ポートの役割と状態

  • ルートポート: ルートブリッジへ最短で向かうポート(各Switchに1つ)
  • 指定ポート: 各セグメントでフレームを転送する側のポート
  • ブロッキングポート: ループを防ぐために転送を止めているポート

状態の確認はこのコマンドが基本です。Root の表示と、各ポートの FWD(転送)/ BLK(ブロック)を読み取ります。

SW1# show spanning-tree

PC接続ポートには PortFast

STP のポートは、有効になってから転送を始めるまで最大50秒ほど待ち時間があります。PC しかつながないアクセスポートではループの心配がないため、PortFast を設定して即座に転送を開始させるのが実務の定石です。

SW1(config)# interface fastEthernet0/1 SW1(config-if)# spanning-tree portfast

PortFast を Switch 間リンクに入れない

PortFast は STP の安全確認を省略する設定です。Switch 同士をつなぐポートに設定すると、一時的にループが発生する危険があります。「PC・サーバーをつなぐポート専用」と覚えてください。

EtherChannel

EtherChannel: 複数リンクを1本の論理リンクに束ねる

STP は冗長リンクをブロックするため、2本つないでも帯域は1本分しか使えません。EtherChannel で複数の物理リンクを1本の論理リンクに束ねると、STP からは1本に見えるためブロックされずに全リンクの帯域を使え、1本切れても残りで通信が続きます。

ネゴシエーションの方式

  • LACP(標準規格): モードは active(自分から交渉)と passive(相手待ち)
  • PAgP(Cisco独自): モードは desirable と auto
  • on: 交渉せず強制的に束ねる(両端とも on のときのみ)

実務では標準規格の LACP を使い、少なくとも片側を active にするのが一般的です。

設定と確認

束ねたいポートをまとめて選択し、同じチャネルグループ番号を割り当てます。速度・デュプレックス・VLAN 設定は束ねる全ポートで一致させる必要があります。

SW1(config)# interface range fastEthernet0/23 - 24 SW1(config-if-range)# channel-group 1 mode active SW1# show etherchannel summary

show etherchannel summary で Po1(SU) のように SU(Layer2・使用中)と表示されれば成功です。

HSRP

HSRP: デフォルトゲートウェイを二重化する

PC のデフォルトゲートウェイに設定した Router が壊れると、PC は外のネットワークへ一切出られなくなります。HSRP は Router 2台で1つの仮想IPアドレスを共有し、PC からは「壊れない1台のゲートウェイ」に見せる Cisco の技術です(標準規格版は VRRP)。

PC-AEnd Device192.168.10.10 /24
GW 192.168.10.254
R1ActiveG0/0: 192.168.10.2
仮想IP 192.168.10.254
R2StandbyG0/0: 192.168.10.3
仮想IP 192.168.10.254

ActiveとStandby

  • 2台のうちプライオリティが高い方(初期値100)が Active になり、仮想IP宛の通信を処理する
  • Standby は Hello パケットで Active を監視し、応答が途絶えると自動で Active に昇格する
  • PC のデフォルトゲートウェイには、実IPではなく仮想IP(この例では 192.168.10.254)を設定する

R1(Active側)の設定

R1(config)# interface gigabitEthernet0/0 R1(config-if)# ip address 192.168.10.2 255.255.255.0 R1(config-if)# standby 1 ip 192.168.10.254 R1(config-if)# standby 1 priority 110 R1(config-if)# standby 1 preempt R1(config-if)# no shutdown

R2 は同じ手順で実IPを 192.168.10.3 にし、priority は初期値のままにします。preempt は「復旧したら Active の座を取り戻す」設定です。

動作確認は show standby brief

R1# show standby brief で、自分の状態(Active / Standby)と仮想IPを一覧できます。R1 の電源を落として PC から Ping を打ち続けると、数秒の断のあと R2 が引き継いで通信が復活する様子を Packet Tracer で観察できます。

Mini Lab

この単元のラボ構成

Switch 2台をケーブル2本でつないで EtherChannel を組み、その上に HSRP の Router 2台を配置します。作ったら、ケーブルを1本抜く・Active の Router を落とすなど「わざと壊す実験」までがこのラボです。

構築の手順

  • 2960 を2台配置し、Fa0/23-24 同士を2本のケーブルで接続する
  • 両方の Switch で channel-group 1 mode active を設定する
  • 1941 を2台配置し、それぞれ SW1・SW2 に接続する
  • R1 に 192.168.10.2、R2 に 192.168.10.3、HSRP の仮想IPを 192.168.10.254 で設定する
  • PC-A(GW 192.168.10.254)から Ping を流しながら障害を起こして切り替わりを観察する

確認コマンドまとめ

  • show spanning-tree: ルートブリッジとポート状態
  • show etherchannel summary: 束ねたリンクの状態
  • show standby brief: HSRP の Active / Standby
  • show standby: プライオリティや仮想MACの詳細

障害試験の前後でこれらの出力を見比べると、「どの仕組みが、どこで切り替えたのか」を根拠を持って説明できるようになります。

Packet Tracer Basics / Hands-on

Packet Tracerの画面と、最初の操作を覚える。

実機に入る前の練習として、Packet Tracerはとても優秀です。ここでは、初心者が最初に触るべき機能だけに絞って説明します。

Unit Topology

この単元で構築する最小構成

Packet Tracer の最初の練習では、End Device 1台、L2 Switch 1台、Router 1台の最小構成を使います。ここで使うIPを先に頭へ入れておくと、以降の操作が理解しやすくなります。

PC-AEnd Device
192.168.10.10 /24
GW 192.168.10.1
SW1Catalyst 2960
管理IP 192.168.10.2 /24
R1Router
G0/0: 192.168.10.1 /24

装置の役割

  • PC-A はIP設定の練習対象
  • SW1 はL2でPCとRouterを中継するSwitch
  • R1 はPC-A のデフォルトゲートウェイになるRouter

値の見方

  • 192.168.10.10 は PC-A 自身のIP
  • 255.255.255.0 は同一ネットワークの範囲
  • 192.168.10.1 は別ネットワークへ出る出口
Workspace Guide

まず見るべき画面のポイント

機器パレット

画面下部付近から Router、Switch、End Devices を選んでワークスペースに配置します。最初は PC、2960系Switch、Router の3種類で十分です。

Realtime と Simulation

Realtime は通常動作、Simulation はパケットの流れを可視化するモードです。概念を掴む段階では Simulation が強力です。

PCのDesktopタブ

PCをクリックすると Desktop タブから IP Configuration を開けます。ここでたとえば 192.168.10.10、255.255.255.0、192.168.10.1 のような具体的な値を入力します。

Router / Switch のCLIタブ

機器をクリックして CLI タブを開くと Cisco IOS 風のコマンド操作ができます。ここが今回の中心です。

Address Example

IP、サブネットマスク、デフォルトゲートウェイを値で理解する

ここでは PC-A に 192.168.10.10 / 255.255.255.0 / 192.168.10.1 を入れる前提で考えます。数字の意味を分けて理解すると、設定ミスがぐっと減ります。

具体例: PC-A に入れる値

  • IPアドレス: 192.168.10.10
  • サブネットマスク: 255.255.255.0
  • デフォルトゲートウェイ: 192.168.10.1

この3つは別々の意味を持っています。特に、IPアドレスとデフォルトゲートウェイを同じ値にしてしまうミスは初心者に多いです。

この値が意味すること

  • 192.168.10.10 は PC-A 自身の住所
  • 255.255.255.0 は「192.168.10」までが同じネットワークだと示す
  • 192.168.10.1 は別ネットワークへ出るための出口で、RouterのIPになる
項目 今回の値 どう見るか
IPアドレス 192.168.10.10 PC-A 自身の識別子。最後の 10 がこの端末を表すイメージです。
サブネットマスク 255.255.255.0 192.168.10.x までが同じセグメントだと判断します。たとえば 192.168.10.1 は同じネットワーク、192.168.20.1 は別ネットワークです。
デフォルトゲートウェイ 192.168.10.1 相手が別ネットワークにいるとき、PC-A が最初に送る相手です。通常はRouterのインターフェースIPを入れます。

数字で考えると分かりやすいポイント

PC-A から 192.168.10.20 に送るなら同じネットワーク内なので直接届く可能性があります。一方、192.168.20.20 に送りたいなら別ネットワークなので、まず 192.168.10.1 のRouterへ渡す必要があります。

First Actions

最初のラボを作るまでの手順

STEP 1

新しいファイルを開く

Packet Tracer を起動し、新規ワークスペースを開きます。最初は白紙から作るほうが構成の意味を理解しやすいです。

STEP 2

PC、Switch、Routerを1台ずつ置く

End Devices から PC、Switches から 2960、Routers から任意のRouterを1台ずつ置きます。

STEP 3

ケーブルで接続する

Connections から自動接続または Copper Straight-Through を選び、PC-Switch、Switch-Router の順に接続します。

STEP 4

PCにIPを設定する

PCの Desktop → IP Configuration を開き、今回は次の値を実際に入れます。値を入れるときは、IPアドレスとデフォルトゲートウェイを混同しないことが大切です。

IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1
STEP 5

RouterのCLIを開く

Routerをクリックし、CLIタブからコマンド入力を始めます。今回は Router 側のLANポートに 192.168.10.1 / 255.255.255.0 を設定する想定です。次のページでこの操作を具体的に学びます。

Mini Lab

このページで作る最小構成

PC-A192.168.10.10 /24
GW 192.168.10.1
SW1VLAN 1: 192.168.10.2 /24
GW 192.168.10.1
R1G0/0: 192.168.10.1 /24

この段階で目指すこと

  • 必要な機器を正しく配置できる
  • ケーブル接続ができる
  • PCのIP設定場所がわかる
  • 最小構成の各機器にどのIPを振るか把握できる
  • RouterとSwitchのCLIタブを開ける

最小構成のアドレス表

  • PC-A: 192.168.10.10 / 255.255.255.0
  • PC-A Default Gateway: 192.168.10.1
  • SW1 VLAN 1 管理IP: 192.168.10.2 / 255.255.255.0
  • SW1 ip default-gateway: 192.168.10.1
  • R1 G0/0: 192.168.10.1 / 255.255.255.0

Switchの管理IPは、学習上「SwitchにもIPを振るとどう見えるか」を理解するために入れています。L2スイッチの通常転送そのものに必須ではありませんが、管理用途でよく使われます。

Mini Config

最小構成で入れる具体的な設定値

ここでは Packet Tracer 上で「実際に何を入れるか」が見えるように、PC、Switch、Routerそれぞれの値を揃えてあります。

PC-A の設定

Desktop > IP Configuration IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1

R1 の設定イメージ

Router> enable Router# configure terminal Router(config)# interface gigabitEthernet0/0 Router(config-if)# ip address 192.168.10.1 255.255.255.0 Router(config-if)# no shutdown

SW1 の管理IP設定イメージ

Switch> enable Switch# configure terminal Switch(config)# interface vlan 1 Switch(config-if)# ip address 192.168.10.2 255.255.255.0 Switch(config-if)# no shutdown Switch(config-if)# exit Switch(config)# ip default-gateway 192.168.10.1

初心者が引っかかりやすい点

Routerのインターフェースは初期状態で shutdown のことがあります。見た目上つながっていても、192.168.10.1 を入れたあとに Router(config-if)# no shutdown をしないと通信できない場合があります。

Troubleshooting

動かないときの最初の確認

接続先ポート

違うポートに差していないか、RouterとSwitchが正しいインターフェースでつながっているかを見ます。

IP設定

PCなら 192.168.10.10 / 255.255.255.0 / 192.168.10.1、Routerなら 192.168.10.1 / 255.255.255.0 になっているか具体値で確認します。

Simulation確認

Packet TracerのSimulationモードで、どこまでパケットが進んだかを追うと原因の切り分けがしやすくなります。

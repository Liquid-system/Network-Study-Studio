Packet Tracer Basics / Hands-on

Packet Tracerの画面と、最初の操作を覚える。

Packet Tracer は、実機に触れる前の練習環境としてとても優秀です。ここでは、初心者が最初に覚えるべき機能だけに絞って説明します。

Unit Topology

この単元で構築する最小構成

最初の練習では、PC 1台・L2 Switch 1台・Router 1台の最小構成を使います。ここで使う IP アドレスを先に頭に入れておくと、このあとの操作の意味が分かりやすくなります。

PC-AEnd Device192.168.10.10 /24
GW 192.168.10.1
SW1Catalyst 2960管理IP 192.168.10.2 /24
R1ISR 1941G0/0: 192.168.10.1 /24

装置の役割

  • PC-A は IP 設定を練習する端末
  • SW1 は PC と Router を L2 で中継する Switch
  • R1 は PC-A のデフォルトゲートウェイになる Router

値の見方

  • 192.168.10.10 は PC-A 自身の IP アドレス
  • 255.255.255.0 は同じネットワークの範囲を示すサブネットマスク
  • 192.168.10.1 は別のネットワークへ出るときの出口(デフォルトゲートウェイ)
Workspace Guide

まず見るべき画面のポイント

機器パレット

画面下部のパレットから Router、Switch、End Devices を選んでワークスペースに配置します。最初は PC、2960 系の Switch、1941 の Router の3種類だけで十分です。機種の選び方は、このあとの「装置の選び方とポート名の関係」で詳しく説明します。

Realtime と Simulation

Realtime は実際の機器と同じように動くモード、Simulation はパケットの流れをゆっくり可視化できるモードです。仕組みを理解する段階では、Simulation モードが特に役立ちます。

PCのDesktopタブ

PC をクリックすると、Desktop タブから IP Configuration を開けます。ここに 192.168.10.10、255.255.255.0、192.168.10.1 といった具体的な値を入力します。

Router / Switch のCLIタブ

機器をクリックして CLI タブを開くと、Cisco IOS と同じ感覚でコマンドを入力できます。この教材の中心になる画面です。

Address Example

IP、サブネットマスク、デフォルトゲートウェイを値で理解する

ここでは PC-A に 192.168.10.10 / 255.255.255.0 / 192.168.10.1 を設定する前提で考えます。それぞれの数字の意味を分けて理解しておくと、設定ミスがぐっと減ります。

具体例: PC-A に入れる値

  • IPアドレス: 192.168.10.10
  • サブネットマスク: 255.255.255.0
  • デフォルトゲートウェイ: 192.168.10.1

この3つはそれぞれ役割が違います。特に、IP アドレスとデフォルトゲートウェイに同じ値を入れてしまうミスは、初心者によく見られます。

この値が意味すること

  • 192.168.10.10 は PC-A 自身の住所
  • 255.255.255.0 は「192.168.10」までが同じネットワークだと示す
  • 192.168.10.1 は別のネットワークへ出るための出口で、R1 の LAN 側 IP を指定する
項目 今回の値 どう見るか
IPアドレス 192.168.10.10 PC-A 自身を表す住所です。最後の 10 がこの端末を指しているイメージです。
サブネットマスク 255.255.255.0 192.168.10.x までが同じネットワークだと判断する目印です。たとえば 192.168.10.1 は同じネットワーク、192.168.20.1 は別のネットワークになります。
デフォルトゲートウェイ 192.168.10.1 宛先が別のネットワークにいるとき、PC-A が最初にデータを渡す相手です。通常は Router のインターフェースの IP を指定します。

数字で考えると分かりやすいポイント

PC-A から 192.168.10.20 に送る場合は、同じネットワーク内なので直接届きます。一方 192.168.20.20 に送りたい場合は別のネットワークなので、まず 192.168.10.1 の Router に渡す必要があります。

Device Selection

装置の選び方とポート名の関係

Packet Tracer のパレットには、Router や Switch の機種が何種類も並んでいます。どれを選んでも学習はできますが、機種によって搭載しているポートの種類と番号が違うため、選んだ機種が教材と異なると、コマンド例のポート名がそのままでは使えません。この教材では、次の機種を選ぶことを前提にしています。

ポートの種類とポート名

Cisco 機器のポート名は「ポートの種類 + 番号」で決まります。代表的な種類は次の2つです。

  • FastEthernet(100Mbps): Fa0/1 のように表記。コマンドでは interface fastEthernet0/1
  • GigabitEthernet(1Gbps): G0/0 のように表記。コマンドでは interface gigabitEthernet0/0

同じ「0/1」でも、FastEthernet 0/1 と GigabitEthernet 0/1 は別のポートです。種類の部分まで含めて初めてポートが特定できます。

機種が違うとポート名も変わる

たとえば古い機種の 1841 は FastEthernet しか搭載していないため、教材の gigabitEthernet0/0 という指定は存在しません。逆に新しい ISR4331 は GigabitEthernet0/0/0 という3階層の番号になり、これも教材の表記と一致しません。

選んだ機種に実際にどんなポートがあるかは、CLI で次のコマンドを打つと一覧できます。

Router# show ip interface brief
役割 この教材で選ぶ機種 搭載ポートと教材での使い方
Router 1941 GigabitEthernet が G0/0・G0/1 の2ポート。教材のコマンド例(interface gigabitEthernet0/0 など)がそのまま使えます。同じポート名を持つ 2911 でも代用できます。
Switch(L2) Catalyst 2960 PC の接続には FastEthernet の Fa0/1〜Fa0/24 を、アップリンクには GigabitEthernet の G0/1・G0/2 を使います。
L3 Switch Catalyst 3560 L3 スイッチ・SVI の単元で使用します。ダウンリンクは 2960 と同じく FastEthernet(Fa0/1〜)です。

別の機種を選んでしまったら

教材と違う機種でも、ポート名を読み替えれば同じ手順で学習できます。たとえば 1841 や 2811 なら interface gigabitEthernet0/0 を interface fastEthernet0/0 に、ISR4331 なら interface gigabitEthernet0/0/0 に読み替えてください。迷ったら show ip interface brief で実際のポート名を確認してから設定するのが確実です。

First Actions

最初のラボを作るまでの手順

STEP 1

新しいファイルを開く

Packet Tracer を起動し、新規ワークスペースを開きます。最初は白紙から自分で組み立てたほうが、構成の意味を理解しやすくなります。

STEP 2

PC、Switch、Routerを1台ずつ置く

End Devices から PC を、Switches から 2960 を、Routers から 1941 を選んで、1台ずつ配置します。Router の機種は必ず 1941(または 2911)にしてください。別の機種を選ぶと、GigabitEthernet ではなく FastEthernet 搭載だったり番号の桁数が違ったりして、教材のポート名と一致しなくなります。

STEP 3

ケーブルで接続する

Connections から自動接続、または Copper Straight-Through を選び、PC と Switch、Switch と Router の順につなぎます。

STEP 4

PCにIPを設定する

PC の Desktop → IP Configuration を開き、次の値を入力します。IP アドレスとデフォルトゲートウェイを取り違えないように注意してください。

IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1
STEP 5

RouterのCLIを開く

Router をクリックし、CLI タブからコマンド入力を始めます。今回は Router の LAN 側ポートに 192.168.10.1 / 255.255.255.0 を設定する想定です。具体的なコマンドは次のページで学びます。

Mini Lab

このページで作る最小構成

PC-A192.168.10.10 /24GW 192.168.10.1
SW1VLAN 1: 192.168.10.2 /24GW 192.168.10.1
R1G0/0: 192.168.10.1 /24

この段階で目指すこと

  • 必要な機器を正しく配置できる
  • ケーブル接続ができる
  • PCのIP設定場所がわかる
  • 最小構成の各機器にどのIPを振るか把握できる
  • RouterとSwitchのCLIタブを開ける

最小構成のアドレス表

  • PC-A: 192.168.10.10 / 255.255.255.0
  • PC-A Default Gateway: 192.168.10.1
  • SW1 VLAN 1 管理IP: 192.168.10.2 / 255.255.255.0
  • SW1 ip default-gateway: 192.168.10.1
  • R1 G0/0: 192.168.10.1 / 255.255.255.0

SW1 の管理 IP は、通信の転送そのものに必須ではありません。ここでは「Switch にも IP を振るとどう見えるか」を体験するために設定しています。実務でも、Switch へリモート接続するための管理用 IP としてよく使われます。

Mini Config

最小構成で入れる具体的な設定値

Packet Tracer 上で「実際に何を入力するのか」が分かるように、PC・Switch・Router それぞれの設定値をまとめました。

PC-A の設定

Desktop > IP Configuration IP Address: 192.168.10.10 Subnet Mask: 255.255.255.0 Default Gateway: 192.168.10.1

R1 の設定イメージ

Router> enable Router# configure terminal Router(config)# interface gigabitEthernet0/0 Router(config-if)# ip address 192.168.10.1 255.255.255.0 Router(config-if)# no shutdown

SW1 の管理IP設定イメージ

Switch> enable Switch# configure terminal Switch(config)# interface vlan 1 Switch(config-if)# ip address 192.168.10.2 255.255.255.0 Switch(config-if)# no shutdown Switch(config-if)# exit Switch(config)# ip default-gateway 192.168.10.1

初心者が引っかかりやすい点

Router のインターフェースは、初期状態では shutdown(無効)になっていることがあります。ケーブルがつながって見えても、192.168.10.1 を設定したあとに Router(config-if)# no shutdown を入力しないと通信できない場合があります。

Troubleshooting

動かないときの最初の確認

接続先ポート

ケーブルを違うポートに差していないか、Router と Switch が意図したインターフェース同士でつながっているかを確認します。

IP設定

PC なら 192.168.10.10 / 255.255.255.0 / 192.168.10.1、Router なら 192.168.10.1 / 255.255.255.0 と、具体的な値どおりに入っているかを確認します。

Simulation確認

Simulation モードでパケットがどこまで進んだかを追いかけると、原因の切り分けがしやすくなります。

Check

この単元の理解度チェック

Packet Tracer のどこを触り、どの値を入れるのかを確認します。
